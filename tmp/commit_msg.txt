fix: module detection regression breaking non-backend modules

## Critical Bug Fixed
find_module_root() in base/scripts/env/paths.py used incorrect module detection
logic (backend/ directory), breaking ux and other non-backend modules. Changed to
use actual module requirements per module_setup.py: pyproject.toml + tests/.

## Problem
Non-backend modules (ux, learning) could not be tested or pushed. When running
git_push.sh ux, tests executed at orchestrator level instead of ux level, discovering
516 tests from ALL modules and failing with import errors.

Root cause: find_module_root() traversed from ux â†’ orchestrator because ux lacks
backend/ directory. pytest then used base/pytest.ini with testpaths=tests relative
to /base/, discovering base/tests/ which includes all submodules.
